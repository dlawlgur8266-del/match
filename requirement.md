# 충북match — 충북대학교 스포츠 매치 시스템 요구사항 명세서

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | **충북match** |
| 목적 | 충북대학교 학생들을 위한 웹 기반 스포츠 매치 플랫폼 |
| 데이터베이스 | Supabase (PostgreSQL + Realtime) |
| 배포 서버 | Vercel |
| 지원 종목 | ⚽ 축구 / 🥅 풋살 / 🏀 농구 / 🎮 e스포츠 |

## 2. 디자인 요구사항

- 스포츠 매치 시스템에 어울리는 **활동적이고 깔끔한 UI/UX**
- 종목별 이모티콘 적용
  - ⚽ 축구
  - 🥅 풋살
  - 🏀 농구
  - 🎮 e스포츠
- 반응형 디자인 (모바일 / 태블릿 / 데스크톱)
- 실시간 알림 표시 영역 (상단 헤더)

---

## 3. 회원가입 (Sign Up)

### 3.1 입력 필드

| 필드 | 유효성 검사 |
|------|------------|
| 아이디 | 타 유저와 중복 불가, 영문+숫자 조합 |
| 비밀번호 | 최소 8자 이상 |
| 비밀번호 확인 | 비밀번호와 일치 여부 검사 |
| 이름 (실명) | 한글 2~5자 |
| 닉네임 | 타 유저와 중복 불가 |
| 학번 | **8자리** 숫자만 허용, 실제 충북대 학번 형식 검증 |

### 3.2 유효성 검사 규칙

- 아이디 중복 확인 버튼 제공 (실시간 Supabase 조회)
- 닉네임 중복 확인 버튼 제공 (실시간 Supabase 조회)
- 비밀번호 ≠ 비밀번호 확인 시 가입 불가 및 오류 메시지 표시
- 학번은 **반드시 8자리**이어야 하며, 입력 제한 8자리 적용
- 모든 필드 입력 완료 후 가입 버튼 활성화

---

## 4. 로그인 (Sign In)

- 아이디 + 비밀번호 입력
- 등록된 계정 정보와 일치할 때만 로그인 허용
- 실패 시 "아이디 또는 비밀번호가 올바르지 않습니다." 오류 메시지 표시
- 로그인 상태는 세션/토큰으로 유지

---

## 5. 네비게이션 메뉴

| 메뉴 | 설명 |
|------|------|
| 매치 | 등록된 매치 목록 조회 및 신청 |
| 매치글 작성 | 새 매치 게시글 작성 |
| 팀 후기 | 매치 완료 후 매너 평가 |
| 내 정보 | 프로필 확인 및 수정 |
| 매치 메시지 | 매치 수락 후 실시간 채팅 |

## 6. 매치 (Match) 기능

### 6.1 매치 목록

- 매치글 작성을 통해 등록된 경기만 표시
- 종목 필터링 기능 (축구 / 풋살 / 농구 / e스포츠)
- 수준 필터링 기능 (초급 / 중급 / 고수)
- 실시간 데이터 반영 (Supabase Realtime)

### 6.2 매치 신청 프로세스

```
신청자: [매치 신청] 버튼 클릭
    ↓
시스템: 매치글 작성자에게 실시간 알림 발송
    ↓
알림 내용: "닉네임 | 실력 수준" 표시
    ↓
작성자 선택
    ├─ [매치 수락] → 매치 확정, 매치 메시지 채널 생성
    └─ [매치 거절] → 신청자에게 "거절되었습니다." 알림 발송
                     매치글은 목록에 계속 유지
```

### 6.3 매치 상태

| 상태 | 설명 |
|------|------|
| 모집 중 | 신청 가능 상태 |
| 매치 확정 | 수락 후 매치 성사 상태 |

---

## 7. 매치글 작성

### 7.1 작성 필드

| 필드 | 옵션 |
|------|------|
| 팀명 | 직접 입력 |
| 종목 선택 | ⚽ 축구 / 🥅 풋살 / 🏀 농구 / 🎮 e스포츠 |
| 매치 인원 | 1vs1 / 3vs3 / 5vs5 / 11vs11 |
| 소개글 | 자유 텍스트 입력 |
| 원하는 수준 | 초급 / 중급 / 고수 |

### 7.2 작성 규칙

- 로그인한 사용자만 작성 가능
- 작성 즉시 매치 목록에 실시간 반영

---

## 8. 팀 후기 (Team Review)

### 8.1 평가 조건

- **매치 수락** 이후, 매치 확정된 경기에 한해 후기 작성 가능
- 매치에 참여한 양측 팀 모두 서로를 평가

### 8.2 평가 방식

| 항목 | 내용 |
|------|------|
| 매너 별점 | ⭐ 1점 ~ ⭐⭐⭐⭐⭐ 5점 |
| 수정 여부 | **불가** (최초 평가 후 수정 금지) |
| 반영 | 평가 완료 시 상대팀의 "내 정보 > 평가" 섹션에 실시간 반영 |

---

## 9. 매치 메시지 (Match Message)

### 9.1 채팅 조건

- 매치 수락이 완료된 경우에만 채팅 채널 생성
- 참여자: 매치글 작성자 ↔ 매치 신청자 (1:1 채팅)

### 9.2 기능 요구사항

| 기능 | 설명 |
|------|------|
| 실시간 채팅 | Supabase Realtime 기반 메시지 즉시 전송/수신 |
| 메시지 기록 | 대화 내역 저장 및 불러오기 |
| 알림 | 새 메시지 수신 시 상단 알림 표시 |

---

## 10. 내 정보 (My Profile)

### 10.1 표시 정보

- 닉네임
- 아이디
- 학번 (마스킹 표시)
- 실력 수준

### 10.2 정보 수정

| 수정 항목 | 설명 |
|-----------|------|
| 닉네임 수정 | 변경 즉시 실시간 반영 (중복 검사 포함) |
| 실력 수준 변경 | 초급 / 중급 / 고수 중 선택 후 즉시 반영 |

### 10.3 평가 섹션

- 받은 매너 별점 목록 표시
- 평균 매너 점수 표시
- 매치별 평가 내역 확인 가능

---

## 11. 알림 시스템

| 알림 유형 | 수신자 | 내용 |
|----------|--------|------|
| 매치 신청 알림 | 매치글 작성자 | "닉네임 님이 매치를 신청했습니다. [실력: OO]" |
| 매치 수락 알림 | 신청자 | "매치가 수락되었습니다!" |
| 매치 거절 알림 | 신청자 | "거절되었습니다." |
| 새 메시지 알림 | 채팅 상대방 | "새 메시지가 도착했습니다." |

- 모든 알림은 Supabase Realtime 기반으로 **실시간** 전달
- 읽지 않은 알림 개수 배지 표시

---

## 12. Supabase 데이터베이스 스키마 (요약)

### 주요 테이블

| 테이블 | 설명 |
|--------|------|
| `profiles` | 사용자 프로필 (닉네임, 학번, 실력 수준 등) |
| `matches` | 매치 게시글 정보 |
| `match_applications` | 매치 신청 정보 및 수락/거절 상태 |
| `reviews` | 팀 후기 및 별점 |
| `notifications` | 알림 내역 |
| `messages` | 매치 메시지 채팅 내역 |

---

## 13. 실시간 요구사항

| 기능 | 실시간 반영 |
|------|------------|
| 매치 목록 | ✅ 새 게시글 즉시 표시 |
| 매치 신청 알림 | ✅ 작성자에게 즉시 전달 |
| 수락/거절 알림 | ✅ 신청자에게 즉시 전달 |
| 채팅 메시지 | ✅ 즉시 전송/수신 |
| 내 정보 수정 | ✅ 변경 즉시 반영 |
| 후기 평가 반영 | ✅ 즉시 상대 프로필에 반영 |

---

## 14. 비기능 요구사항

- **보안**: 인증되지 않은 사용자의 매치 신청, 글 작성 차단 (Row Level Security)
- **성능**: Vercel Edge Network 기반 빠른 응답
- **확장성**: Supabase 무료 플랜 → 필요 시 유료 플랜 확장 가능
- **접근성**: 모든 주요 기능은 로그인 후 접근 가능
